1. PRESENTATION GENERALE
--------------------------------------------------------------------------------

SPARKY est une application web full-stack developpee dans le cadre d'un Projet
de Fin d'Annee, destinee aux entreprises du secteur photovoltaique en Tunisie.

L'application automatise la gestion, l'analyse et la comparaison des documents
commerciaux (factures, devis, bons de livraison, bons de commande, avoirs)
grace a l'integration de l'intelligence artificielle generative (Mistral AI),
d'une base de donnees relationnelle (PostgreSQL), d'une orchestration de
workflows (n8n) et d'une interface utilisateur moderne (Angular).

OBJECTIFS METIER :
  - Reduire le temps de saisie manuelle des factures fournisseurs
    (de ~5 minutes a ~8 secondes par document)
  - Centraliser les documents commerciaux dans une base structuree
  - Comparer objectivement les offres de plusieurs fournisseurs
  - Suivre les niveaux de stock automatiquement a partir des bons de livraison
  - Notifier l'equipe par email a chaque scan avec le detail extrait

--------------------------------------------------------------------------------
4. FONCTIONNALITES DEVELOPPEES
--------------------------------------------------------------------------------

4.1 AUTHENTIFICATION SECURISEE
  - Endpoint POST /auth/login (verification bcrypt)
  - Endpoint POST /auth/register (creation utilisateur)
  - Stockage des mots de passe haches (algorithme bcrypt + salt)
  - Persistance de session cote frontend (localStorage)
  - Roles utilisateurs : user / admin

4.2 SCANNER DE DOCUMENTS
  - Import par glisser-deposer ou selection de fichier
  - Capture par camera (mobile-friendly)
  - Formats supportes : PDF, JPG, PNG, TIFF
  - Pipeline IA en temps reel (~8 secondes par document)
  - Affichage du score de confiance IA (eleve / moyen / faible)

4.3 CLASSIFICATION AUTOMATIQUE
  - Detection du type : facture / devis / bon de livraison /
    bon de commande / avoir
  - Score de confiance numerique (0.0 a 1.0)
  - Modele IA : mistral-small-latest (PDF) ou pixtral-12b (image)

4.4 EXTRACTION STRUCTUREE DES DONNEES
  Champs extraits automatiquement :
  - Identite fournisseur (nom, SIRET, contact)
  - Numero de document
  - Dates (emission, echeance, validite, livraison)
  - Montants HT, TVA, TTC, devise
  - Lignes articles : reference, designation, quantite,
    unite, prix unitaire, remise, taux TVA, montant ligne

4.5 PERSISTANCE POSTGRESQL
  - Sauvegarde transactionnelle apres chaque scan reussi
  - Modele relationnel strict (16 tables avec contraintes)
  - Endpoint GET /documents avec JOIN multi-tables
  - Statut du document : pending / classified / extracted / error

4.6 TABLEAU DE BORD
  - Statistiques par type de document
  - Listing des derniers documents scannes
  - Actions rapides : scanner, comparer, voir documents
  - Affichage du fournisseur, numero, montant TTC, score IA

4.7 GESTION DES DOCUMENTS
  - Liste de tous les documents scannes
  - Filtrage par type et recherche par fournisseur ou numero
  - Tri par date, montant, type, nom
  - Suppression individuelle
  - Detail complet : lignes, totaux, score IA

4.8 COMPARATEUR DE PRIX MULTI-FOURNISSEURS
  - Import de plusieurs devis / factures de fournisseurs differents
  - Extraction automatique des articles (sans saisie manuelle)
  - Tableau croise articles x fournisseurs
  - Mise en evidence du meilleur prix par article
  - Calcul du total TTC par fournisseur (PU x Qte x remise x TVA)
  - Identification du fournisseur le moins cher

4.9 CATALOGUE DE STOCK
  - Vue centralisee des prix par article et par fournisseur
  - Niveau de stock courant (calcule par SUM des mouvements)
  - Historique des entrees / sorties
  - Lien automatique entre articles, fournisseurs, documents et mouvements

4.10 EXPORT MULTI-CANAL
  - Export Excel (openpyxl) avec mise en forme
  - Export Google Sheets (1 ligne par article extrait)
  - Notification email HTML detaillee (tableau articles + bouton sheet)

--------------------------------------------------------------------------------
6. PIPELINE D'ORCHESTRATION (n8n)
--------------------------------------------------------------------------------

Le backend, apres extraction reussie, declenche un POST sur le webhook n8n.
Le workflow execute en chaine :

  1. Reception Document       (webhook entree)
  2. Validation et Detection  (PDF / image, modele IA utilise)
  3. Classification IA        (label francais, confiance %, niveau)
  4. Resultat Classification  (transit)
  5. Extraction Donnees IA    (lignes, totaux, moteur extracteur)
  6. Resultat Extraction      (transit)
  7. Preparation lignes Sheet (1 row par article : 16 colonnes)
  8. Export Google Sheet      (append-row sur sheet partage)
  9. Construction Email HTML  (template responsive avec tableau styled)
 10. Envoi Email SMTP         (vers le mail configure)
 11. Reponse Finale           (acknowledgement JSON)


--------------------------------------------------------------------------------
7. SCHEMA DE BASE DE DONNEES
--------------------------------------------------------------------------------

Le schema PostgreSQL contient 16 tables organisees autour de 3 axes :

7.1 AUTHENTIFICATION
  - users                 (id, email, password_hash, nom, role, created_at)

7.2 DOCUMENTS COMMERCIAUX
  - fournisseurs          (annuaire des fournisseurs)
  - documents             (table maitre, tous types confondus)
  - factures              (donnees specifiques aux factures)
  - devis                 (donnees specifiques aux devis)
  - bons_commande         (donnees specifiques aux bons de commande)
  - bons_livraison        (donnees specifiques aux BL)
  - avoirs                (donnees specifiques aux avoirs)
  - lignes_facture        (detail articles facture)
  - lignes_devis          (detail articles devis)
  - lignes_bl             (detail articles BL)

7.3 CATALOGUE ET STOCK
  - articles              (catalogue normalise + specifications JSONB)
  - articles_fournisseurs (prix achat / vente par fournisseur)
  - mouvements_stock      (entrees/sorties, stock = SUM(quantite))

ENUMS PostgreSQL :
  - document_type  : facture / devis / bon_commande / bon_livraison / avoir
  - document_status: pending / classified / extracted / error
  - mouvement_type : entree / sortie / inventaire / retour


--------------------------------------------------------------------------------
8. SECURITE
--------------------------------------------------------------------------------

  - Hashage des mots de passe avec bcrypt (salt + cost factor 12)
  - Validation des emails par EmailStr (RFC 5322)
  - Validation des entrees par Pydantic (typage strict)
  - Variables sensibles dans .env (Mistral key, DB password, n8n secret)
  - CORS configure (whitelist origins)
  - Webhook n8n protege par header X-Sparky-Secret
  - SQL parametre (resistance aux injections)


--------------------------------------------------------------------------------
9. RESULTATS ET BENEFICES
--------------------------------------------------------------------------------

GAINS QUANTIFIES :
  - Temps moyen de saisie d'une facture :
      avant : 5 minutes (saisie manuelle ligne par ligne)
      apres : 8 secondes (scan + IA)
      => Gain : 97% de reduction du temps

  - Score moyen de confiance IA observe : 85-92%
  - Capacite de traitement : ~7 documents par minute en simultane
  - Persistance fiable (PostgreSQL ACID, pas de perte de donnees)

BENEFICES METIER :
  - Comparaison objective et instantanee des offres fournisseurs
  - Tracabilite complete (qui a scanne quoi, quand, avec quel resultat)
  - Reduction des erreurs de saisie humaine
  - Centralisation des documents dans une base requetable
  - Notifications temps reel (email + Google Sheets) sans intervention humaine
